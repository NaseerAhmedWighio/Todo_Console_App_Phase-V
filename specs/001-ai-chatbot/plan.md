## Summary

The Todo AI Chatbot feature implements a natural language interface that allows users to manage their tasks through conversational interactions. The system uses OpenAI's Assistant API with MCP (Model Context Protocol) tools to enable the AI agent to perform task operations (add, list, update, complete, delete) while maintaining strict separation of responsibilities as required by the constitution.

The implementation extends the existing FastAPI backend with new conversation and message models stored in the PostgreSQL database. A new chat endpoint handles authentication, loads conversation history, passes user messages to the AI agent, and persists the agent's responses. The MCP server exposes task operations as standardized tools that the AI agent can call based on user intent, ensuring all database interactions occur through proper tool interfaces rather than direct access.

## Technical Context

**Language/Version**: Python 3.11+ (based on existing backend infrastructure)
**Primary Dependencies**: FastAPI, SQLModel, Neon PostgreSQL, httpx (for OpenRouter API), MCP Server Framework
**Storage**: Neon Serverless PostgreSQL (existing infrastructure)
**Testing**: pytest (to be integrated with existing test framework)
**Target Platform**: Linux server (web-based AI chatbot service)
**Project Type**: Web application (extension of existing backend)
**Performance Goals**: <3 second response time for chat interactions, 90%+ intent recognition accuracy
**Constraints**: Stateless backend requirement (no in-memory conversation state), MCP tool integration, JWT authentication enforcement
**Scale/Scope**: Support for multiple concurrent users with isolated conversations and tasks

**III. Tool-Only Task Access**: ✓ COMPLIANT

**IV. Authentication Enforcement**: ✓ COMPLIANT

- Agent operates within authenticated user scope only
- Unauthenticated requests will be rejected with HTTP 401

**V. Separation of Responsibilities**: ✓ COMPLIANT
- Agent decides intent
- MCP tools perform actions
- Database persists state
- API orchestrates flow
