# Todo AI Chatbot - Phase III Constitution

## Core Principles

### I. Spec-Driven Development Only
No manual coding is allowed; all implementation must be generated by Claude Code; If output is incorrect, the spec must be refined — not the code; All implementation artifacts must be produced through AI-assisted generation rather than hand-written code.

### II. Stateless Backend
The FastAPI server must hold no in-memory state; All conversation and task state must persist in the database; Backend services must be designed for horizontal scaling without relying on server-side session storage; System must maintain functionality across server restarts and deployments.

### III. Tool-Only Task Access
AI agents must NEVER access the database directly; All task operations must be performed via MCP tools only; Agents interact with system data exclusively through defined API endpoints and tool interfaces; Direct database queries by agents are strictly prohibited.

### IV. Authentication Enforcement
Every request must be authenticated using JWT from Better Auth; Agents operate strictly within the authenticated user scope; All API requests must verify JWT tokens with appropriate HTTP 401 responses for unauthenticated access; Authentication checks must be implemented at the API gateway or middleware level.

### V. Separation of Responsibilities
Agent decides intent; MCP tools perform actions; Database persists state; API orchestrates flow; Clear boundaries must be maintained between AI decision-making, tool execution, data persistence, and API orchestration layers.

### VI. OpenRouter API Integration
The system must use OpenRouter API with the xiaomi/mimo-v2-flash:free model for cost-effective AI processing; All API calls must use the OPENROUTER_API_KEY for authentication; The system must handle OpenRouter API responses and tool calling functionality appropriately.

## Safety & Boundaries
Agent must not hallucinate information or generate false data; All responses must be grounded in actual system state or provided context; AI agents must acknowledge uncertainty rather than fabricating information; Strict adherence to user privacy and data protection requirements.

## Development Standards

All code must follow clean, readable conventions appropriate for the technology stack; Use appropriate design patterns for maintainability and extensibility; Implement proper error handling, logging, and user feedback mechanisms; All components must be properly tested with comprehensive test coverage; Security considerations must be prioritized in all implementations.

## Development Workflow

Follow the spec → plan → tasks workflow strictly; Each feature must have clear acceptance criteria defined before implementation; Code reviews required for all changes with adherence to constitutional principles; All tests must pass before merging; Changes must be verified against all constitutional principles during review; MCP tool integrations must be validated for all new functionality.

## Governance

This constitution governs all development practices for the Todo AI Chatbot Phase III. All PRs and reviews must verify compliance with these principles; Any deviation from these principles must be justified and approved; Architecture decisions must align with the specified technology stack and design patterns; All code must be generated through Claude Code as mandated; Compliance reviews must verify stateless architecture, tool-only access, and proper authentication enforcement.
